## 📂 BIMIPAL PROJECT CARD (Context for AI)

**Опис:**
Telegram-бот для ведення сімейного бюджету (Вадим та Аня). Дані зберігаються в Google Sheets. Проєкт розрахований на роботу 24/7 на робочому ПК (Windows), розробка ведеться на ноутбуці.

### 🛠 Технічний стек
* **Мова:** Python 3.x
* **Бібліотеки:**
    * `aiogram 3.x` (асинхронний бот)
    * `gspread` (робота з таблицями)
    * `oauth2client` (авторизація Google)
    * `matplotlib` (генерація діаграм)
    * `apscheduler` (планувальник завдань/нагадувань)
    * `re` (RegEx для очищення даних)
* **База даних:** Google Sheets (Service Account).
* **Хостинг:** Локальний робочий ПК (Windows 10/11) у режимі 24/7.
* **Версійність:** Git (GitHub Private Repo).

---

### 🗂 Структура файлів
* `main.py` — Основний файл. FSM, хендлери, планувальник (`scheduler`), меню.
* `gs_manager.py` — Клас `GoogleSheetManager`. Методи для роботи з обома аркушами (Транзакції та Планування).
* `visuals.py` — Модуль для генерації графіки (кругові діаграми з легендою).
* `config.py` — **(.gitignore)** Налаштування, токени, ID таблиці, списки категорій.
* `service_account.json` — **(.gitignore)** Ключ доступу до Google Cloud API.
* `Run_Bot.bat` — Скрипт-сторож. Автозапуск, авто-оновлення (`git pull`) та рестарт.
* `.gitignore` — Ігнорує секрети (`config.py`, `.json`, `venv`).

### 📊 Структура Google Таблиці
Таблиця містить два основні аркуші:

**1. Аркуш "Транзакції" (Лог операцій)**
Бот додає нові записи у **3-й рядок**.
* **A:** Дата
* **B:** Категорія
* **C:** Сума
* **D:** Тип ("Витрати"/"Поповнення")
* **E:** Опис
* **F:** Примітка
* **G:** Хто

**2. Аркуш "Планування" (Бюджетні ліміти)**
Використовується для контролю витрат.
* **A:** Категорія (напр. "🛒 Продукти")
* **B:** Ліміт (напр. "6000")

### 🔄 Workflow розробки (DevOps)
**1. Середовище розробки (DEV):**
* Ноутбук | Гілка `dev` | Бот `@bimipaltest_bot` | Аркуш `"Тест"`.

**2. Продакшн (PROD):**
* Робочий ПК | Гілка `main` | Бот `@bimipal_bot` | Аркуш `"Транзакції"`.

**Алгоритм релізу:**
1. Код в `dev` -> Тест -> Merge в `main` -> Push.
2. На робочому ПК команда `/restart` (або `Run_Bot.bat`) оновлює код.
*(Примітка: При додаванні нових бібліотек потрібне ручне оновлення `pip install -r requirements.txt`).*

### ✅ Поточний функціонал (v2.0)

**💰 Облік:**
1.  **Швидка витрата:** Сума -> Опис (Категорія "⏳ Очікує уточнення").
2.  **Детальний облік:** Витрати та Доходи за категоріями.
3.  **Скасування:** Кнопка "↩️ Скасувати" (видаляє останній запис).

**👮‍♂️ Контроль бюджету (Планування):**
1.  **Ліміти в хмарі:** Бот читає ліміти з аркуша "Планування".
2.  **Миттєвий алерт:** При запису витрати бот перевіряє ліміт. Якщо перевищено — попереджає (`⚠️ УВАГА! Переліміт...`).
3.  **Редагування:** Кнопка "⚙️ Змінити ліміт" дозволяє змінювати бюджет категорії прямо в чаті.
4.  **Перевірка:** Кнопка "🔔 Перевірити ліміти" показує звіт з індикаторами (🟢🟡🔴).

**📊 Аналітика та UX:**
1.  **Діаграми:** Кнопка "🍰 Діаграма витрат" (Donut chart з легендою праворуч).
2.  **Звіти:** Статистика за поточний місяць та за останні 7 днів.
3.  **UX:** Відображення статусу "Друкує..." або "Відправляє фото" при обробці запитів.
4.  **Admin:** Команда `/restart` для дистанційного оновлення.

### 🗺 ROADMAP v3.0 (Стратегія розвитку)

#### 🟢 Етап 3.0: "Порядок і Баланс" (Priority: High)
* **Глобальний баланс:** Відображення "Всього грошей" (Wallet) разом із "Місячним потоком" (Cashflow).
* **Оновлення категорій:** Додавання `🏠 Оренда`, `⚡️ Комуналка`, `🌐 Інтернет/ТБ`, `💵 Купівля валюти`.
* **Початкова точка:** Внесення корегуючого запису для синхронізації реального балансу.

#### 🟡 Етап 3.1: "Валютний сейф" (USD Wallet)
* **Мультивалютність:** Окремий облік доларових заощаджень.
* **Операції:**
    * "Купив 100$" -> Списання грн, зарахування USD.
    * "Зняв 50$" -> Списання USD, запис приходу грн та їх витрати.
    * Обов'язковий коментар при обміні.

#### 🟠 Етап 3.2: "Особистий секретар" (Нагадування)
* **Аркуш "Нагадування":** Таблиця з датами (Оренда, Інтернет, Підписки).
* **Логіка:** Щоденна перевірка дати -> Повідомлення -> Кнопка `[✅ Сплатив]`.
* **Smart Tracking:** Якщо не оплачено — нагадування переноситься на наступний вечір.

#### 🔴 Етап 3.3: "Повний контроль" (Редагування)
* **Історія:** Команда `/history` (останні 5-10 записів).
* **Inline-кнопки:** `[✏️ Ред.]` та `[🗑 Вид.]` під кожним записом.
* **Безпека:** Додаткове підтвердження при видаленні або зміні лімітів.
