📂 BIMIPAL PROJECT CARD (Context for AI)

Опис: Telegram-бот для ведення сімейного бюджету (Вадим та Аня). Дані зберігаються в Google Sheets. Проєкт розрахований на роботу 24/7 на робочому ПК (Windows), розробка ведеться на ноутбуці.

🛠 Технічний стек

    Мова: Python 3.x

    Бібліотеки:

        aiogram 3.x (асинхронний бот)

        gspread (робота з таблицями)

        oauth2client (авторизація Google)

    База даних: Google Sheets (Service Account).

    Хостинг: Локальний робочий ПК (Windows 10/11) у режимі 24/7.

    Версійність: Git (GitHub Private Repo).

🗂 Структура файлів

    main.py — Основний файл запуску бота (FSM, хендлери, клавіатури).

    gs_manager.py — Клас GoogleSheetManager для запису даних у таблицю.

    config.py — (.gitignore) Налаштування, токени, ID таблиці, списки категорій.

    service_account.json — (.gitignore) Ключ доступу до Google Cloud API.

    Run_Bot.bat — Скрипт для автозапуску, авто-оновлення (git pull) та перезапуску при падінні.

    .gitignore — Ігнорує секрети (config.py, .json, venv).

📊 Структура Google Таблиці

Бот додає нові записи у 3-й рядок (Insert Row), зберігаючи заголовки. Стовпчики:

    A: Дата (dd.mm.yyyy)

    B: Категорія

    C: Сума (float)

    D: Тип операції ("Витрати" / "Поповнення")

    E: Що купив (Опис)

    F: Примітка (Технічне поле, напр. "Telegram", "Швидкий запис")

    G: Хто (Ім'я користувача)

🔄 Workflow розробки (DevOps)

У нас налаштовано розділення середовищ:

1. Середовище розробки (DEV):

    Де: Ноутбук.

    Гілка Git: dev.

    Бот: Тестовий (@bimipaltest_bot).

    Таблиця: Аркуш "Тест".

    Config: У локальному config.py прописаний токен тестового бота.

2. Продакшн (PROD):

    Де: Робочий ПК.

    Гілка Git: main.

    Бот: Основний (@bimipal_bot).

    Таблиця: Аркуш "Транзакції".

    Автоматизація: Файл Run_Bot.bat при кожному запуску робить git pull origin main, оновлюючи код.

Алгоритм внесення змін:

    Кодимо в гілці dev на ноуті -> Пуш в origin dev.

    Тестуємо.

    Зливаємо в реліз: git checkout main -> git merge dev -> git push origin main.

    На робочому ПК бот оновлюється сам при перезапуску.

⚙️ Конфігурація (config.py)

Файл містить:

    BOT_TOKEN: (Різний для Dev і Prod).

    SPREADSHEET_ID: ID гугл таблиці.

    SHEET_NAME: "Транзакції" (Prod) або "Тест" (Dev).

    USERS: Dict {ID: "Imya"} (Вадим, Аня).

    EXPENSE_CATEGORIES / INCOME_CATEGORIES: Списки категорій для кнопок.

✅ Поточний функціонал

    Швидка витрата: Введення суми -> Введення опису -> Автоматично категорія "⏳ Очікує уточнення".

    Детальна витрата/Дохід: Вибір типу -> Вибір категорії (кнопки) -> Сума -> Опис.

    UI: Кнопки ReplyKeyboardMarkup (persistent), захист від випадкових натискань (фільтр станів), підтримка Emoji.
