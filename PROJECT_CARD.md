## 📂 BIMIPAL PROJECT CARD (Context for AI)

**Опис:**
Telegram-бот для ведення сімейного бюджету (Вадим та Аня). Дані зберігаються в Google Sheets. Проєкт розрахований на роботу 24/7 на робочому ПК (Windows), розробка ведеться на ноутбуці.

### 🛠 Технічний стек
* **Мова:** Python 3.x
* **Бібліотеки:**
    * `aiogram 3.x` (асинхронний бот)
    * `gspread` (робота з таблицями)
    * `oauth2client` (авторизація Google)
    * `re` (RegEx для очищення сум)
* **База даних:** Google Sheets (Service Account).
* **Хостинг:** Локальний робочий ПК (Windows 10/11) у режимі 24/7.
* **Версійність:** Git (GitHub Private Repo).

---

### 🗂 Структура файлів
* `main.py` — Основний файл запуску. Містить FSM, обробку команд `/start`, `/restart`, кнопок меню.
* `gs_manager.py` — Клас `GoogleSheetManager`. Методи: додавання (`add_transaction`), статистика (`get_month_stats`), скасування (`undo_last_transaction`).
* `config.py` — **(.gitignore)** Налаштування, токени, ID таблиці, списки категорій.
* `service_account.json` — **(.gitignore)** Ключ доступу до Google Cloud API.
* `Run_Bot.bat` — Скрипт-сторож (Watchdog). Забезпечує автозапуск, авто-оновлення (`git pull`) та перезапуску при падінні/рестарті.
* `.gitignore` — Ігнорує секрети (`config.py`, `.json`, `venv`).

---

### 📊 Структура Google Таблиці
Бот додає нові записи у **3-й рядок** (Insert Row), зберігаючи заголовки.
**Стовпчики:**
* **A:** Дата (dd.mm.yyyy)
* **B:** Категорія
* **C:** Сума (float) — *очищується від "грн", пробілів та ком через RegEx.*
* **D:** Тип операції ("Витрати" / "Поповнення")
* **E:** Що купив (Опис)
* **F:** Примітка (Технічне поле, напр. "Telegram", "Швидкий запис")
* **G:** Хто (Ім'я користувача)

### 🔄 Workflow розробки (DevOps)
У нас налаштовано розділення середовищ:

**1. Середовище розробки (DEV):**
* **Де:** Ноутбук.
* **Гілка Git:** `dev`.
* **Бот:** Тестовий (`@bimipaltest_bot`).
* **Таблиця:** Аркуш `"Тест"`.
* **Config:** У локальному `config.py` прописаний токен тестового бота.

**2. Продакшн (PROD):**
* **Де:** Робочий ПК.
* **Гілка Git:** `main`.
* **Бот:** Основний (`@bimipal_bot`).
* **Таблиця:** Аркуш `"Транзакції"`.
* **Автоматизація:** Файл `Run_Bot.bat` при кожному циклі робить `git pull origin main`.

**Алгоритм внесення змін:**
1.  Кодимо в гілці `dev` на ноуті -> Пуш в `origin dev`.
2.  Тестуємо функціонал.
3.  Зливаємо в реліз: `git checkout main` -> `git merge dev` -> `git push origin main`.
4.  На робочому ПК (або телефоні) пишемо `/restart` — бот оновлюється сам.

---

### ⚙️ Конфігурація (`config.py`)
Файл містить:
* `BOT_TOKEN`: (Різний для Dev і Prod).
* `SPREADSHEET_ID`: ID гугл таблиці.
* `SHEET_NAME`: "Транзакції" (Prod) або "Тест" (Dev).
* `USERS`: Dict `{ID: "Imya"}` (Вадим, Аня).
* `EXPENSE_CATEGORIES` / `INCOME_CATEGORIES`: Списки категорій.

---

### ✅ Поточний функціонал
1.  **Швидка витрата:** Введення суми -> Введення опису -> Автоматично категорія "⏳ Очікує уточнення".
2.  **Детальна витрата/Дохід:** Вибір типу -> Вибір категорії (кнопки) -> Сума -> Опис.
3.  **Статистика:** Кнопка "📊 Статистика". Показує витрати/дохід/баланс за поточний місяць + Топ-3 категорії. (Українська локалізація місяців).
4.  **Скасування:** Кнопка "↩️ Скасувати". Видаляє останній доданий запис (рядок 3) з таблиці.
5.  **Remote Restart:** Команда `/restart`. Завершує процес бота (`sys.exit`), що тригерить `Run_Bot.bat` на оновлення коду та перезапуск.
6.  **UI:** Persistent клавіатура, захист від випадкових натискань (StateFilter).
