=== Шпаргалка розробника (Cheat Sheet). ===

>> 🛠 Частина 1: Розробка (Гілка dev)
Робимо це вдома, коли п'ємо каву і пишемо новий код.
    Переконайся, що ти в гілці розробки (має світитися dev):

-----------------------------------------------
git checkout dev
-----------------------------------------------

    Змінюй код, тести через py main.py.

    Зберігай зміни в хмару (щоб не загубити):

-----------------------------------------------
git add .
git commit -m "Опис того, що ти змінив"
git push origin dev
-----------------------------------------------

    (Це не впливає на робочого бота, це просто твоя чернетка в інтернеті).

>> 🚀 Частина 2: Реліз (Викочуємо в Продакшн)
Робимо це, коли все протестовано і працює ідеально.
    Переходимо в головну гілку:

-----------------------------------------------
git checkout main
-----------------------------------------------

    (На всяк випадок) Оновлюємо її:

-----------------------------------------------
git pull origin main
-----------------------------------------------

    Зливаємо зміни з чернетки (dev) у чистовик (main):

-----------------------------------------------
git merge dev
-----------------------------------------------

    Відправляємо на сервер (GitHub):

-----------------------------------------------
git push origin main
-----------------------------------------------

    Повертаємося назад у dev для подальшої роботи:

-----------------------------------------------
    git checkout dev
-----------------------------------------------

Як тільки ти зробиш пункт 4, робочий бот побачить оновлення і перезапуститься сам.

>> 🚑 Частина 3: Аварійний відкат
Якщо залив оновлення в main, і бот на роботі впав.

Спосіб 1: Скасувати останній коміт (Безпечно) Це створить "анти-коміт", який скасує зміни.

-----------------------------------------------
git checkout main
git revert HEAD
git push origin main
-----------------------------------------------

(Після цього треба пофіксити помилку в dev і залити заново).

Спосіб 2: Жорсткий відкат (Якщо все зовсім погано) Повертає стан коду рівно на 1 крок назад і стирає помилку з історії.

-----------------------------------------------
git checkout main
git reset --hard HEAD~1
git push --force origin main
-----------------------------------------------
